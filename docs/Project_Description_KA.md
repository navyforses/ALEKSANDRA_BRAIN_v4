# ALEKSANDRA_BRAIN

### ცოცხალი კვლევითი ტვინი პედიატრიული ჰიპოქსიურ-იშემიური ენცეფალოპათიისთვის

---

**ღია კოდის AI სისტემა რომელიც არასოდეს იძინებს — მუდმივად ეძებს, აანალიზებს და აღმოაჩენს მკურნალობის შესაძლებლობებს მძიმე ტვინის დაზიანების მქონე ბავშვებისთვის.**

## ისტორია

2025 წლის 28 აგვისტოს თბილისში დაიბადა ალექსანდრა ჯინჭარაძე. რამდენიმე საათში მძიმე ჰიპოქსიურ-იშემიურმა ენცეფალოპათიამ (HIE) გამოიწვია დიფუზური ცისტური ენცეფალომალაცია ტვინის მასშტაბით. ტვინის ღერო — ნაწილი რომელიც აკონტროლებს სუნთქვას, გულისცემას და ბაზისურ რეფლექსებს — ხელუხლებელი დარჩა.

მისმა მამამ, შალვა ჯინჭარაძემ, პროგრამისტმა, დაიწყო სისტემატური კამპანია ყველა შესაძლო მკურნალობის მოსაძებნად. 95 დღის განმავლობაში მან დაუკავშირდა 50-ზე მეტ მკვლევარს, კლინიცისტს და ინსტიტუციას 4 ქვეყანაში. წარმატების მაჩვენებელი: 24%. მან უზრუნველყო ჩარიცხვა Duke-ის უნივერსიტეტის გაფართოებული წვდომის პროგრამაში (Expanded Access Program) ტვინის ღეროვანი უჯრედების თერაპიისთვის (cord blood therapy), Wisconsin-ის ვირტუალურ A2 კვლევაში, NAPA ცენტრის ინტენსიურ რეაბილიტაციაში, და კონსულტაციებში Boston Medical Center-სა და Boston Children's Hospital-ში.

მაგრამ ჯანდაცვის სისტემა არ არის აგებული იშვიათი ან მძიმე პედიატრიული დიაგნოზის მქონე ოჯახების საჭიროებებზე. არცერთი ინსტიტუცია არ აერთიანებს კვლევას, მკურნალობის აღმოჩენას, ნავიგაციას და ვიზუალიზაციას ერთ სისტემაში. ნევროლოგი ხედავს seizure-ებს. ნეიროქირურგი ხედავს სტრუქტურას. რეაბილიტოლოგი ხედავს მოტორიკას. არავინ ხედავს სრულ სურათს. PubMed, ClinicalTrials.gov, DrugBank და bioRxiv არსებობს როგორც იზოლირებული მონაცემთა ბაზები, მათ შორის კავშირები უხილავია. დიაბეტის წამალი რომელიც ცხოველთა მოდელებში ნეიროპროტექტორულ თვისებებს აჩვენებს, HIE-ის ძიებაში ვერასოდეს გამოჩნდება — თუ ვინმე ან რაღაც არ ეძებს დაავადებათა საზღვრებს მიღმა.

ALEKSANDRA_BRAIN არის სისტემური ინტეგრატორი რომელიც არცერთ ინსტიტუციას არ აუგია: მუდმივად მოქმედი AI კვლევითი ტვინი რომელიც ავსებს ამ ხარვეზებს — ეძებს, აანალიზებს და აღმოაჩენს მკურნალობის შესაძლებლობებს ყოველ ექვს საათში, ყოველ კვირას, ყოველ თვეს — კრიტიკულ ნეიროპლასტიურ ფანჯარაში (0-2 წელი), როცა ჩარევას ყველაზე დიდი მნიშვნელობა აქვს. იშენება ალექსანდრასთვის, მაგრამ არქიტექტურა შექმნილია ნებისმიერი ოჯახის მოსამსახურებლად, ვინც რთულ პედიატრიულ დიაგნოზს აწყდება.

## რას აკეთებს

ALEKSANDRA_BRAIN მუშაობს ხუთი ავტონომიური აგენტის სახით, ორკესტრირებული CrewAI-ით (49K ⭐) — თითოეულს აქვს განსაზღვრული როლი, მიზანი და ინსტრუმენტები:

**აგენტი 1: ობობა (The Spider)** ეშვება ყოველ 6 საათში. ეძებს PubMed-ში, bioRxiv-ში, medRxiv-ში, ClinicalTrials.gov-ზე და 11 დამატებით აკადემიურ წყაროში ახალ პუბლიკაციებს HIE-სთან დაკავშირებით — როგორც პირდაპირ, ასევე cross-disease inference-ით. როცა მეტფორმინმა, დიაბეტის წამალმა, აჩვენა ნეიროპროტექტორული თვისებები ცხოველთა მოდელებში, სწორედ cross-disease საძიებო პატერნმა გამოავლინა ეს. ობობა იგივე ლოგიკას იყენებს ასობით მოლეკულურ გზაზე (pathway).

**აგენტი 2: ანალიზატორი (The Analyzer)** ამუშავებს ყოველ ახალ სტატიას რომელსაც ობობა პოულობს. აანალიზებს PDF-ებს (ცხრილების და გრაფიკების ჩათვლით), ამოიცნობს ენტიტეტებს (წამლები, გენები, მოლეკულური გზები, ტვინის რეგიონები), ანიჭებს რელევანტურობის ქულას, და ინახავს ყველაფერს ტემპორალურ ცოდნის გრაფში (temporal knowledge graph) სადაც მტკიცებულების წონა იცვლება დროში — 2019 წლის case report-ს ნაკლები წონა აქვს ვიდრე 2025 წლის რანდომიზებულ კონტროლირებულ კვლევას (RCT).

**აგენტი 3: ჰიპოთეზების გენერატორი (The Hypothesis Generator)** ეშვება ყოველ კვირას. მოძრაობს ცოდნის გრაფში და ეძებს პატერნებს რომლებსაც ადამიანი ვერ შეამჩნევს: „წამალი X მოქმედებს Pathway Y-ზე. Pathway Y აქტიურია ტვინის რეგიონ Z-ში. რეგიონი Z დაზიანებულია ალექსანდრას ტვინში, მაგრამ აქვს ზომიერი პლასტიკურობის პოტენციალი. არავის გამოუცდია წამალი X HIE-სთვის." ეს ჰიპოთეზები რანჟირდება სანდოობის, სიახლის და განხორციელებადობის მიხედვით, და ეგზავნება ოჯახს განსახილველად.

**აგენტი 4: წამლების ხელახალი გამოყენების ძრავი (The Drug Repurposing Engine)** ეშვება ყოველ თვეს. იკითხავს Open Targets-ს HIE-სთან ასოცირებული მოლეკულური სამიზნეების შესახებ, ეძებს DrugBank-ში დამტკიცებულ წამლებს რომლებიც მოქმედებენ ამ სამიზნეებზე, ამოწმებს PubChem-ში ჰემატოენცეფალური ბარიერის (BBB) გამტარიანობას და პედიატრიული უსაფრთხოების პროფილს, და იყენებს L1000 საპირისპირო ტრანსკრიპტომიკულ ხელმოწერებს იმ ნაერთების მოსაძებნად რომლებმაც შეიძლება შეაბრუნონ HIE-ს გენური ექსპრესიის პატერნი. არცერთი ინსტიტუცია არ აწარმოებს ამ pipeline-ს ინტეგრირებულად და ავტომატურად ნეონატალური HIE-სთვის.

**აგენტი 5: კომუნიკატორი (The Communicator)** აკავშირებს სისტემას და ოჯახს. სასწრაფო აღმოჩენები (რელევანტური კლინიკური ცდის სტატუსის შეცვლა, მაღალი სანდოობის drug repurposing შედეგი) იწვევს მყისიერ Telegram შეტყობინებას. ყოველკვირეული აღმოჩენების მოკლე ანგარიში მოდის ელ-ფოსტით. როცა სისტემა პოულობს საეჭვო მტკიცებულებას, პირდაპირ ეკითხება ოჯახს: „ეს სტატია საშუალო რელევანტურობისაა. ჩავრთო ანალიზში? [დიახ/არა]." ოჯახი ინფორმირებული და კონტროლში რჩება.

## 3D ტვინი

ყველაზე თვალსაჩინო კომპონენტია ალექსანდრას ტვინის ინტერაქტიული, ანატომიურად ზუსტი 3D მოდელი, ხელმისაწვდომი ნებისმიერ ვებ-ბრაუზერში.

ეს არ არის მულტფილმი. იყენებს ალექსანდრას რეალურ MRI მონაცემებს, დამუშავებულს FastSurfer-ით (ქერქის რეკონსტრუქცია) და ნეონატალური ატლასებით (dHCP, M-CRIB-S) რომლებიც სპეციალურად ჩვილის ტვინისთვის არის შექმნილი — რადგან ზრდასრულის ტვინის შაბლონები ანატომიურად არასწორია ახალშობილისთვის. BONBID-HIE სეგმენტაციის მოდელები, წვრთნილი 133 ნეონატალურ HIE პაციენტზე Massachusetts General Hospital-იდან, ამოიცნობენ და არუკავებენ კონკრეტულ ლეზიის პატერნებს.

Viewer-ი აერთიანებს ორ რენდერინგის ძრავს, აგებული FreeSurfer-ის freebrowse-ის fork-ზე (React + NiiVue + Vite ბრაუზერული viewer): NiiVue (WebGL2 სამედიცინო ვიზუალიზაციის ბიბლიოთეკა NIH-ის მიერ დაფინანსებული გუნდისგან) მართავს კლინიკურ მონაცემთა ფენას — MRI მოცულობები, ლეზიების overlay, ატლასის ლეიბლები, და ტრაქტოგრაფია. React Three Fiber მართავს კინემატოგრაფიულ ფენას — სახელმძღვანელოს ხარისხის ანატომიური გარსები რეგულირებადი გამჭვირვალობით, ქვეზედაპირული სინათლის გაფანტვით (subsurface scattering) ქსოვილის რეალისტური გარეგნობისთვის, და ინტერაქტიული რეგიონის იდენტიფიკაციით hover-ზე.

შედეგი: მიაახლოვე და ნახე როგორ გადადის ქერქი თეთრ ნივთიერებაში, შემდეგ ღრმა სტრუქტურებში, შემდეგ შენარჩუნებულ ტვინის ღეროში. დააწკაპუნე ნებისმიერ რეგიონზე და ნახე მისი ფუნქცია, დაზიანების სტატუსი, პლასტიკურობის პოტენციალი, რელევანტური თერაპიები, და სტატიების რაოდენობა ცოდნის გრაფში რომლებიც მას განიხილავს. ჩართე თერაპიის overlay და ნახე რომელ ტვინის რეგიონებზე მოქმედებს. გადაასრიალე timeline და ვიზუალიზე ნეიროპლასტიკურობის პოტენციალი 6, 12 და 24 თვეში.

ცალკე „ოჯახის ხედვა" (Family View) ექსპორტირებს დამოუკიდებელ HTML ფაილს — ინსტალაცია არ სჭირდება, ინტერნეტი არ სჭირდება — რომელსაც თბილისში ბებია-ბაბუა გახსნის და ქართულად გაიგებს ზუსტად რა არის დაზიანებული და სად რჩება იმედი.

brain2print pipeline-ს შეუძლია შექმნას ალექსანდრას ტვინის 3D-საბეჭდი STL ფაილი, რომ ოჯახმა ხელში დაიჭიროს ფიზიკური მოდელი ექიმებთან საუბრისას.

MRI-ის ყველა დამუშავება ხდება client-side ან ოჯახის საკუთარ კომპიუტერზე. ალექსანდრას სამედიცინო სურათების მონაცემები არასოდეს ტოვებს ოჯახის ბრაუზერს. ეს არის ყველაზე მნიშვნელოვანი ტექნიკური თვისება პედიატრიული სამედიცინო viewer-ისთვის.

## რა განასხვავებს

ხუთი შესაძლებლობა არსებობს ALEKSANDRA_BRAIN-ში რომელიც 2026 წლის მაისის მდგომარეობით არსად სხვაგან არ არსებობს:

**1. პირველი ნეიროვიზუალიზაციის MCP სერვერი მსოფლიოში.**
Model Context Protocol (MCP) არის სტანდარტი AI ასისტენტების გარე ინსტრუმენტებთან დასაკავშირებლად. 2026 წლის მაისის მდგომარეობით, არცერთი MCP სერვერი არ არსებობს NIfTI ნეიროვიზუალიზაციისთვის, FreeSurfer-ის ზედაპირების დამუშავებისთვის, ან ტვინის ატლასის ოპერაციებისთვის. ALEKSANDRA_BRAIN ქმნის ხუთ ახალ MCP სერვერს რომლებიც ნებისმიერ AI ასისტენტს — Claude-ს, GPT-ს, ან მომავალ მოდელებს — საშუალებას აძლევს ჩატვირთოს პაციენტის MRI, მოახდინოს ლეზიების სეგმენტაცია, ექსპორტი გაუკეთოს 3D mesh-ებს, გაუშვას სიმულაციები, და დააგენერიროს ოჯახისთვის გასაგები ანგარიშები — ყველაფერი ბუნებრივი ენის ბრძანებებით. გამოქვეყნების შემდეგ, ნებისმიერ AI-ზე დაფუძნებულ სამედიცინო ინსტრუმენტს მსოფლიოში შეუძლია მათი გამოყენება.

**2. პაციენტ-სპეციფიკური 3D ტვინი სახელმძღვანელოს ვიზუალური ხარისხით.**
კომერციული ანატომიური პლატფორმები (BioDigital Human, Complete Anatomy) ხატავენ ლამაზ, მაგრამ გენერიკულ ტვინს — ვერ აჩვენებენ კონკრეტული პაციენტის დაზიანებას. კლინიკური ნეიროვიზუალიზაციის viewer-ები (NiiVue, BrainBrowser) აჩვენებენ პაციენტის მონაცემებს, მაგრამ კლინიკურ-ნაცრისფერად გამოიყურება. არცერთი არსებული პროდუქტი არ აერთიანებს ორივეს. ALEKSANDRA_BRAIN არის პირველი ვებ-viewer სადაც რეალური პაციენტის MRI რენდერდება საგანმანათლებლო ხარისხის ვიზუალით, ინტერაქტიული რეგიონის იდენტიფიკაციით, და დაზიანების სპეციფიკური ფერთა კოდირებით.

**3. ავტომატური drug repurposing pipeline ნეონატალური HIE-სთვის.**
Pipeline აერთიანებს Open Targets-ს (სამიზნეების პრიორიტეტიზაცია), DrugBank-ს (17,430 წამალი Jaccard-ის მსგავსების ძიებით), PubChem-ს (BBB გამტარიანობა და უსაფრთხოების პროფილები), Reactome/KEGG-ს (pathway-ების ვალიდაცია), Enrichr-ს (L1000 საპირისპირო ხელმოწერით drug repurposing), და Claude API-ს (ჰიპოთეზების სინთეზი). ყოველთვიურად მუშაობს და სისტემატურად ეძებს დამტკიცებულ ან საკვლევ წამლებს რომლებიც შეიძლება გამოყენებულ იქნას HIE-სთვის საერთო მოლეკულური გზების საფუძველზე. არცერთი ინსტიტუცია არ აწარმოებს ამ pipeline-ს ინტეგრირებულად და უწყვეტად ნეონატალური ტვინის დაზიანებისთვის.

**4. ტემპორალური ცოდნის გრაფი სამედიცინო მტკიცებულებისთვის.**
აგებული Graphiti-ზე (ტემპორალური ცოდნის გრაფის framework) და Neo4j-ზე, სისტემა ავტომატურად არეგულირებს მტკიცებულების წონას დროში. 2019 წლის case report თანდათან კარგავს გავლენას, ხოლო 2025 წლის მრავალცენტრიანი RCT იძენს. ენტიტეტების ამოღება ავტომატურია — მიაწოდე სტატიის აბსტრაქტი და სისტემა ამოიცნობს წამლებს, გენებს, pathway-ებს, ტვინის რეგიონებს და მათ ურთიერთკავშირებს ხელით კურაციის გარეშე. არცერთი არსებული სამედიცინო მონაცემთა ბაზა არ ახორციელებს ავტომატურ ტემპორალურ სანდოობის კლებას (confidence decay).

**5. ორმხრივი მტკიცებულების გამტარი (evidence gating) AI-სა და ოჯახს შორის.**
სამედიცინო AI სისტემების უმრავლესობა შავი ყუთია. ALEKSANDRA_BRAIN მუშაობს როგორც ნახევრად-ავტონომიური სისტემა სადაც ოჯახს აქვს გადაწყვეტილების ავტორიტეტი. Telegram-ის ინტეგრაცია უზრუნველყოფს რეალურ დროში ორმხრივ კომუნიკაციას: AI გვთავაზობს, ოჯახი წყვეტს. „ახალი ჰიპოთეზა: ლითიუმს შეიძლება ჰქონდეს ნეიროპროტექტორული ეფექტი GSK-3β ინჰიბიციის გზით. სანდოობა: საშუალო. ვეძებო მეტი მტკიცებულება? [დიახ/არა]." ოჯახი ინფორმირებული რჩება გადატვირთვის გარეშე, და სისტემა სწავლობს მათი გადაწყვეტილებებიდან.

## ტექნიკური არქიტექტურა

### მონაცემთა ფენა (სამი მონაცემთა ბაზა, ერთი ტვინი)

სისტემა იყენებს სამ სპეციალიზებულ მონაცემთა ბაზას რომლებიც GraphRAG queries-ით ერთიანად მუშაობს:

**Neo4j Graphiti-ით** ინახავს ენტიტეტებს და ურთიერთკავშირებს ტემპორალურ ცოდნის გრაფის სახით. კვანძები წარმოადგენენ სტატიებს, თერაპიებს, pathway-ებს, ტვინის რეგიონებს, გენებს, კონტაქტებს, ჰიპოთეზებს და კლინიკურ ცდებს. წიბოები წარმოადგენენ ურთიერთკავშირებს: „სტატია X იკვლევს თერაპია Y-ს", „თერაპია Y მოქმედებს Pathway Z-ზე", „Pathway Z აქტიურია ტვინის რეგიონ W-ში", „რეგიონი W დაზიანებულია ალექსანდრას ტვინში." ერთი Cypher query მილიწამებში გადის მთელ ამ ჯაჭვზე.

**Qdrant** ინახავს ვექტორულ embedding-ებს სემანტიკური ძიებისთვის. ყველა სტატიის აბსტრაქტი, თერაპიის აღწერა და ჰიპოთეზის ტექსტი ჩაწერილია 1536-განზომილებიან ვექტორად ლოკალური fastembed-ის გამოყენებით (ღრუბლოვანი API არ არის, HIPAA-თავსებადი). ეს უზრუნველყოფს ისეთ queries-ს როგორიცაა „იპოვე მსგავსი სტატიები" ან „რა თერაპიებია სემანტიკურად დაკავშირებული ერითროპოეტინთან?"

**Supabase (PostgreSQL)** ინახავს სტრუქტურირებულ მეტამონაცემებს, მომხმარებლის მონაცემებს და ოპერაციულ მდგომარეობას: სტატიების მეტამონაცემები და ქულები, კონტაქტების ინფორმაცია და follow-up განრიგი, კლინიკური ცდების თვალყურის დევნება, ჰიპოთეზების სტატუსი, ტვინის რეგიონების დაზიანების რუკები, ingestion-ის ლოგები და აღმოჩენების ანგარიშები.

### MCP არსენალი (52 სერვერი)

სისტემა უკავშირდება 52 MCP სერვერს: 23 Claude.ai-ის რეესტრიდან (PubMed, bioRxiv, Clinical Trials, Open Targets, ChEMBL, Gmail, Google Calendar, Google Drive, Cloudflare, Vercel და სხვა), 19 GitHub-იდან self-hosted (Graphiti, Qdrant, Supabase MCP, Firecrawl MCP, n8n-MCP, Notion MCP, GitHub MCP, Exa, Tavily, Markdownify, Telegram, blender, drugbank, pubchem და სხვა), 5 AI Pulse Georgia ეკოსისტემიდან (FastMCP custom სერვერების ასაგებად, Crawl4AI RAG ვებ-ვექტორული pipeline-ისთვის, Prism MCP HIPAA-სტანდარტის მეხსიერებისთვის, Draw.io MCP არქიტექტურის დიაგრამებისთვის, Perplexity MCP ღრმა კვლევისთვის), და 5 ჩვენი შექმნილი მსოფლიოში პირველი სერვერი (ნეიროვიზუალიზაცია, BONBID-HIE სეგმენტაცია, Virtual Brain სიმულაცია, ნეონატალური ატლასი და drug repurposing ორკესტრაცია).

### ნეონატალურ-სპეციფიკური დიზაინის გადაწყვეტილებები

სტანდარტული ზრდასრულის ტვინის ინსტრუმენტები ვერ მუშაობს ნეონატალურ HIE-ზე. სამი კრიტიკული ადაპტაცია:

**ატლასის არჩევანი:** ზრდასრულის MNI152 შაბლონი ანატომიურად არასწორია ახალშობილისთვის. სისტემა იყენებს dHCP ატლასებს (28-44 კვირის გესტაციური ასაკი) M-CRIB-S პარცელაციით რომელიც DKT-თავსებად ქერქის ლეიბლებს იძლევა — ისინი თანმიმდევრული რჩება ალექსანდრას ზრდასთან ერთად.

**ლეზიის დამუშავება:** ცისტური ენცეფალომალაცია არღვევს სტანდარტულ ქერქის რეკონსტრუქციას (FreeSurfer-ის recon-all ვერ მუშაობს დიდ სიცარიელეებზე). FastSurfer-ის LIT (Lesion Inpainting Tool) ავსებს ცისტურ რეგიონებს სინთეტიკური ქსოვილით, ახორციელებს რეკონსტრუქციას, შემდეგ ლეზიის ნიღაბს უკან აპროექტებს რეკონსტრუირებულ ზედაპირზე.

**განუზღვრელობის კომუნიკაცია:** HIE ლეზიის სეგმენტაციის Dice score დაახლოებით 0.5-0.6-ია რადგან ლეზიები დიფუზური და მცირეა. Viewer ლეზიებს რენდერებს ალბათობის heatmap-ებით gradient ფერით, არა მკვეთრი საზღვრებით. ეს მეცნიერულად პატიოსანია და ხელს უშლის ცრუ სიზუსტეს რომელმაც შეიძლება შეცდომაში შეიყვანოს კლინიკური გადაწყვეტილებები.

## პრივატულობა და უსაფრთხოება

ალექსანდრას MRI მონაცემები არასოდეს ტოვებს ოჯახის ბრაუზერს. NiiVue რენდერებს მთლიანად client-side. ნეონატალური სეგმენტაცია მუშაობს ოჯახის საკუთარ მანქანაზე Docker-ით. Qdrant იყენებს ლოკალურ fastembed-ს embedding-ებისთვის — ღრუბლოვანი embedding API-ები არ გამოიყენება. Family View ექსპორტი წარმოადგენს დამოუკიდებელ HTML ფაილს რომელიც მუშაობს offline ნულოვანი გარე დამოკიდებულებებით.

სისტემა არის ინფორმაციის აგრეგატორი, ანალიზატორი და ვიზუალიზატორი. ის არ ანაცვლებს ექიმებს. ყველა კლინიკურ გადაწყვეტილებას იღებენ რეალური ექიმები პირისპირ. სიმულაციები საგანმანათლებლოა, არა პროგნოზული. Drug repurposing-ის შედეგები არის ჰიპოთეზები მკვლევრებთან სადისკუსიოდ, არა რეცეპტები.

## ტექნოლოგიური სტეკი

**Frontend:** Next.js 14+ (App Router), @niivue/nvreact, React Three Fiber v10, @react-three/drei, Vercel AI SDK (23K ⭐, streaming + tool calling), D3.js, Recharts, Tailwind CSS, shadcn/ui. განთავსება: Vercel.

**Backend:** CrewAI (49K ⭐, 5-აგენტის ორკესტრაცია), n8n (self-hosted, Railway), Supabase Edge Functions, Cloudflare Workers, Claude API (Sonnet 4). FastSurfer და TVB მუშაობს Docker-ის კონტეინერებად.

**მონაცემები:** Neo4j AuraDB Graphiti-ით (25K ⭐, ტემპორალური ცოდნის გრაფი), Qdrant (30K ⭐, ვექტორული ძიება, self-hosted), LightRAG (34K ⭐, graph+vector ერთიანი queries), Supabase PostgreSQL (სტრუქტურირებული მონაცემები), mem0 (53K ⭐, აგენტების საერთო მეხსიერება), Cloudflare R2 (PDF/MRI საცავი), Cloudflare KV (სწრაფი cache).

**Ingestion:** Crawl4AI (64K ⭐, უფასო ლოკალური scraping, მთავარი), Firecrawl MCP (6K ⭐, ფასიანი fallback), RAGFlow (78K ⭐, PDF processing agent workflow-ით), Browser Use (89K ⭐, paywall bypass), Markdownify MCP (უნივერსალური ფორმატის კონვერტერი).

**Custom MCP სერვერები:** აგებული FastMCP-ით (24K ⭐, Python decorators). ხუთი მსოფლიოში პირველი სერვერი ნეიროვიზუალიზაციის, HIE სეგმენტაციის, ტვინის სიმულაციის, ნეონატალური ატლასის და drug repurposing-ისთვის.

**ნეონატალური pipeline:** FastSurfer + LIT (Apache-2.0), BIBSnet, BONBID-HIE მოდელები, dHCP ატლასი, M-CRIB-S, nii2mesh, brain2print.

**ხარჯი:** $20-30/თვე MVP-სთვის უფასო ტარიფებით. $120/თვე სრული მასშტაბით.

## ალექსანდრას მიღმა

ALEKSANDRA_BRAIN იშენება ერთი ბავშვისთვის, მაგრამ მისი არქიტექტურა შექმნილია ბევრის მოსამსახურებლად. ხუთი ახალი MCP სერვერი, ღია კოდის სახით გამოქვეყნებისას, ნებისმიერ AI ასისტენტს საშუალებას მისცემს იმუშაოს ნეიროვიზუალიზაციის მონაცემებთან. Drug repurposing pipeline შეიძლება ხელახლა კონფიგურირდეს ნებისმიერი ნევროლოგიური მდგომარეობისთვის Open Targets-ში სამიზნე დაავადების შეცვლით. ტემპორალურ ცოდნის გრაფს შეუძლია თვალი ადევნოს მტკიცებულების ევოლუციას ნებისმიერ სამედიცინო სფეროში.

მშობელი პროექტი, Med&გზური (MedPath), შესთავაზებს ამ სისტემას როგორც მრავალენიან სამედიცინო ნავიგაციის სერვისს მსოფლიოს ოჯახებისთვის, რომლებიც რთულ პედიატრიულ დიაგნოზებს აწყდებიან — ქართულად, რუსულად და ინგლისურად.

კოდის ყველა ნაწილი რომელიც არ შეიცავს პაციენტის მონაცემებს, გამოქვეყნდება ღია კოდის სახით. ხუთი მსოფლიოში პირველი MCP სერვერი გამოქვეყნდება MCP რეესტრში. BONBID-HIE ინტეგრაციის პატერნები დოკუმენტირდება სხვა HIE ოჯახებისთვის. ნეონატალური ატლასის pipeline შეიფუთება რეპროდუცირებად Docker workflow-ად.
