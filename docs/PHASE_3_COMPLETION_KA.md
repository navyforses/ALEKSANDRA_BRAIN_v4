## შენგან საჭიროა ორი მოქმედება

### 1. Gmail OAuth ერთჯერადი დაყენება (CGM-04 → GREEN)

**~15 წუთის სამუშაო, ერთხელ** — შემდეგ ავტომატური ხდება.

ნაბიჯები ნახე [docs/RUNBOOK-gmail-api.md](RUNBOOK-gmail-api.md)-ში.
მოკლედ:

1. Google Cloud Console-ში პროექტი → Gmail API-ის ჩართვა → OAuth client
   credentials JSON-ის ჩამოტვირთვა
2. ფაილი დაასახელე `.secrets/gmail_oauth_credentials.json`-ად (უკვე
   `.gitignore`-შია)
3. ერთჯერადი ბრძანების გაშვება — გაიხსნება ბრაუზერი, დაადასტურე ნებართვა
   (**მხოლოდ "compose drafts" scope** — *ვერც კი ფიქრობს* გაგზავნაზე)
4. Token ფაილი ლოკალურად ინახება

**უსაფრთხოება:** OAuth scope-ი მკაცრად შეზღუდულია `gmail.compose`-ით.
სკრიპტი **ვერ** აგზავნის ემაილს, **ვერ** კითხულობს inbox-ს, **ვერ**
ცვლის ვერაფერს — მხოლოდ Drafts-ში წერს. იცი წერდე, ვერც წერდე — სხვა
არაფერი.

### 2. Notion-დან CSV-ის გადმოცემა (CGM-10 → GREEN)

**~5 წუთის სამუშაო, ერთხელ.**

1. Notion-ში გახსენი კონტაქტების database-ი
2. Export → CSV
3. ფაილი დადე `data/notion_contacts.csv`-ად
4. გაუშვი:
   ```powershell
   .\.venv\Scripts\python.exe -X utf8 -m scripts.import_contacts_from_notion `
       --input data/notion_contacts.csv --dry-run
   ```
   ეს გაჩვენებს რამდენი მწკრივი იქნება import-ი. გადახედე ციფრს.
5. თუ ციფრი მისაღებია (≥75 მოსალოდნელია), გაუშვი `--confirm`:
   ```powershell
   .\.venv\Scripts\python.exe -X utf8 -m scripts.import_contacts_from_notion `
       --input data/notion_contacts.csv --confirm
   ```

**უსაფრთხოება:** ყველა იმპორტირებული კონტაქტი იღებს **მაქსიმალურად დაცულ**
ნაგულისხმევებს — `consent_full_name=FALSE`, `consent_doctor_names=FALSE`,
`consent_hospital_names=FALSE`. შენ ცალკე გადააქცევ თითოეულ კონტაქტს
„ნდობით" კონტექსტში მათ ნებართვის შემდეგ.
