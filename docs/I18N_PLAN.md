# ALEKSANDRA_BRAIN: ბილინგვური (I18N) სისტემის იმპლემენტაციის გეგმა

**სტატუსი:** დაგეგმილი
**თარიღი:** 2026-05 (მიმდინარე სპრინტი)
**მიზანი:** სისტემის (საიტი და მონაცემები) სრული მხარდაჭერა ინგლისურ (`en`) და ქართულ (`ka`) ენებზე.

---

## არქიტექტურული ხედვა
1. **სტატიკური ტექსტები:** იმართება ლოკალური `.json` ლექსიკონებით ფრონტენდში (`next-intl`).
2. **დინამიური მონაცემები:** ინახება Supabase-ში, როგორც `JSONB` ტიპის ველები `{"en": "...", "ka": "..."}` ფორმატით.
3. **ავტომატური თარგმანი:** AI აგენტები გენერირებისთანავე ქმნიან და ბაზაში წერენ ორივე ენის ვერსიას.

---

## ეტაპი 1: Frontend-ის სტატიკური თარგმანი (Next.js App Router)
**მიზანი:** URL-ის მარშრუტიზაცია (Routing) და ინტერფეისის თარგმნა.

**ნაბიჯები:**
1. **ბიბლიოთეკის დამატება:** `npm install next-intl` პროექტის `viewer` ფოლდერში.
2. **სტრუქტურის შეცვლა:**
   - `app/` ფოლდერის შიგთავსის გადატანა `app/[locale]/` ფოლდერში (მაგ: `/en/dashboard`, `/ka/dashboard`).
3. **კონფიგურაციის ფაილები:**
   - `i18n.ts` - ლოკალიზაციის ინიციალიზაცია.
   - `middleware.ts` - მომხმარებლის ბრაუზერის ენის ავტომატური ამოცნობა და სწორ URL-ზე გადაყვანა (Default: `en`).
4. **ლექსიკონების შექმნა:**
   - `messages/en.json` (Dashboard, Timeline, Save, Cancel, ა.შ.)
   - `messages/ka.json` (მართვის პანელი, ქრონოლოგია, შენახვა, გაუქმება, ა.შ.)
5. **ენის გადამრთველი (Language Switcher):**
   - UI კომპონენტის (Header-ში) შექმნა `[EN | GE]` ღილაკებით, რომელიც URL-ს შეცვლის ენის შესაბამისად.

---

## ეტაპი 2: Supabase მონაცემთა ბაზის მოდიფიკაცია (JSONB)
**მიზანი:** დინამიური ცხრილების მომზადება ორენოვანი მონაცემების მისაღებად.

**ნაბიჯები:**
1. **ცხრილების იდენტიფიკაცია:**
   ძირითადად შესაცვლელია ტექსტური ველები შემდეგ ცხრილებში:
   - `aleksandra_timeline` (title, description)
   - `hypotheses` (title, description, evidence_summary)
   - `therapies` (name, status_reason)
   - `briefs` (content)
2. **SQL მიგრაციის სკრიპტი (`scripts/migrations/012_i18n_jsonb.sql`):**
   ნაცვლად `TEXT` ტიპისა, ეს ველები უნდა გახდეს `JSONB`.
   *მაგალითი მექანიზმის:*
   ```sql
   ALTER TABLE aleksandra_timeline
   ALTER COLUMN title TYPE JSONB
   USING jsonb_build_object('en', title, 'ka', title);
   ```
   *(ეს მიდგომა უზრუნველყოფს, რომ ძველი ინგლისური ტექსტები არ დაიკარგოს და დროებით ქართულ ველშიც ჩაჯდეს, სანამ AI გადათარგმნის).*

---

## ეტაპი 3: AI აგენტების და n8n-ის განახლება
**მიზანი:** ახალი მონაცემების ავტომატური გენერაცია ორივე ენაზე.

**ნაბიჯები:**
1. **Prompt-ების განახლება (CrewAI & LangChain):**
   აგენტებს (მაგ. *Hypothesis Agent*, *Communicator Agent*) უნდა დაემატოთ მკაცრი Output Schema.
   *მაგალითი ინსტრუქციის:*
   `"You must return the summary in JSON format with two keys: 'en' for English medical terminology, and 'ka' for the Georgian translation tailored for the family."`
2. **Python სკრიპტების ვალიდაცია:**
   Pydantic მოდელების განახლება (Backend-ში), რათა მიიღონ `dict` ტიპი (en და ka მნიშვნელობებით) ნაცვლად `str` (სტრიქონისა).
3. **Telegram/Gmail შეტყობინებები:**
   n8n workflow-ების მოდიფიკაცია ისე, რომ ოჯახის (Telegram) არხში გაიგზავნოს `data.title.ka`, ხოლო ექიმების (Gmail Draft) არხში — `data.title.en`.

---

## ეტაპი 4: მონაცემთა ეკრანზე გამოტანა (Frontend Binding)
**მიზანი:** Next.js-მა წაიკითხოს JSONB ბაზიდან და გამოაჩინოს ენის მიხედვით.

**ნაბიჯები:**
1. **მონაცემთა გამოთხოვა (Fetch):**
   ფრონტენდში შემოდის ობიექტი `title: { en: "...", ka: "..." }`.
2. **კომპონენტის ლოგიკა:**
   გამოვიყენებთ `next-intl`-ის მიმდინარე ენის სტატუსს (locale).
   ```tsx
   const locale = useLocale(); // 'en' ან 'ka'
   const displayTitle = event.title[locale] || event.title['en']; // Fallback ინგლისურზე
   ```
3. **UI კომპონენტების რეფაქტორინგი:** Timeline, Dashboard, და Therapies გვერდების განახლება ამ ახალი ლოგიკით.

---

## მიმდევრობა (Execution Order)
რეკომენდებულია გეგმის ზუსტად ამ მიმდევრობით შესრულება:
1. ჯერ ეტაპი 1 (Frontend სტატიკური ენა) - ყველაზე უსაფრთხოა და არ ეხება ბაზას.
2. მერე ეტაპი 2 (DB მიგრაცია) + ეტაპი 4 (Frontend დინამიური მონაცემები) - კეთდება ერთად.
3. ბოლოს ეტაპი 3 (AI აგენტები) - რათა ახალი ინფორმაცია უკვე სწორი სტრუქტურით ჩაიწეროს.
