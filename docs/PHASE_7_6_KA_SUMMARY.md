# ფაზა 7.6 — საიტის რეფაქტორი (ოჯახისთვის)

> ეს დოკუმენტი აღწერს იმას, რაც დღევანდელი სამუშაოს შემდეგ გამოჩნდება ALEKSANDRA_BRAIN-ის ვებგვერდზე, როცა შაკო `npm run dev`-ს გაუშვებს ლოკალურად ან Vercel preview აიწყობა.

---

## რა შეიცვალა

4 ახალი გვერდი დაემატა საიტს, 4 არსებული გვერდი გაუმჯობესდა მცირე ვიჯეტებით.

### ახალი გვერდები

**1. ციფრული ტყუპის სტატუსი (`/ka/twin`)**

ალექსანდრას მდგომარეობის 13 განზომილების მიმოხილვა ერთ ბადეზე. თითო ბარათი აჩვენებს:
- განზომილების სახელს (მაგ. "გულყრის სიხშირე", "კუნთთა ტონუსი", "ნეიროპლასტიკურობის ფანჯარა")
- მიმდინარე საშუალო მნიშვნელობას
- 80% სანდოობის ინტერვალს (ანუ რა ფარგლებში ვართ დარწმუნებული)
- პატარა ჰისტოგრამას რომელიც ხატავს ალბათობის განაწილებას
- სამეცნიერო წყაროს (PMID) ციტირებას

ეს არის ერთი მზერით „სად ვართ ალექსანდრას მდგომარეობის გაგებაში" სურათი.

**2. რწმენის დრიფტი (`/ka/drift`)**

დროითი ხედი — ბოლო 30 დღეში როგორ შეიცვალა თითო განზომილების ჩვენი შეფასება. 13 ხაზი ერთ გრაფიკზე, თითო ფერით, რომბის ფორმის ნიშნულები აღნიშნავს დღეებს როცა ახალი მტკიცებულება დაემატა და რწმენა განახლდა. შესაძლებელია თითო განზომილების ჩართვა/გამორთვა.

**3. კაუზალური გრაფი (`/ka/causal`)**

სტრუქტურული კაუზალური მოდელის ვიზუალური ხედი — რომელი ცვლადი იწვევს რომელს, რომელი თრგუნავს, რომელი შუამავლობს. წიბოს ფერი აღწერს კავშირის ტიპს:
- მწვანე: იწვევს (CAUSES)
- წითელი: თრგუნავს (INHIBITS)
- ლურჯი: შუამავლობს (MEDIATES)
- ქარვისფერი: გვერდიდან ერევა (CONFOUNDS)
- იისფერი: არბილებს (MODERATES)

კვანძზე დაკლიკებისას მარჯვენა პანელი აჩვენებს ამ კვანძის მშობლებს, შვილებს, მექანიზმის აღწერას და სამეცნიერო წყაროებს. საცდელად ჩატვირთულია „ვიგაბატრინი → გულყრის სიხშირე" საცნობარო მოდელი 5 კვანძით + 6 წიბოთი.

**4. სიმულაციის სტუდია (`/ka/simulate`)**

ექიმისთვის. სცენარის კონსტრუქტორი (drag-and-drop) საშუალებას აძლევს ააწყოს „რა მოხდება თუ" შეკითხვა: ჩარევები, შედეგები, გამრევი ფაქტორები, შუამავლები. „გაშვების" დაჭერის შემდეგ მონაცემები იგზავნება სიმულატორზე და მარჯვენა პანელი აჩვენებს A vs B სცენარების შედარებას — ჰისტოგრამები გვერდიგვერდ, p(A > B) ნიშნული, ვერდიქტი (A უკეთესია / B უკეთესია / ფრე / ბუნდოვანი). შედეგი არ ნიშნავს მკურნალობას — გადაწყვეტილების უფლება ალექსანდრას ექიმებთან რჩება.

### განახლებული გვერდები

**5. სტატუსის კოკპიტი (`/ka`)** — ზემოთ დაემატა ციფრული ტყუპის სურათი (13 განზომილების კომპაქტური მიმოხილვა + პატარა sparkline ყველაზე გაურკვეველი განზომილებისთვის).

**6. ჰიპოთეზები (`/ka/hypotheses`)** — თითო ჰიპოთეზის ბარათში დაემატა მინი-ჰისტოგრამა მოსალოდნელი სარგებლის განაწილებისთვის (როცა სიმულაცია მიბმულია).

**7. სტატიების ხედი (`/ka/papers`)** — დაემატა „ტყუპზე ზემოქმედების სორტირება" ფილტრი. ეს ფილტრი დაალაგებს სტატიებს იმის მიხედვით, თუ რამდენად შეცვლის თითო სტატია ციფრული ტყუპის შეფასებას (KL divergence).

**8. დღევანდელი ხედი (`/ka/today`)** — დაემატა „ამ კვირის აქტიური კითხვების" სექცია. აქ ჩანს ის შეკითხვები, რომლებიც სისტემამ კვირის განმავლობაში დასვა (მაგ. „რამდენ წამს იჭერს ალექსანდრა თავს ვერტიკალურად?") და მათი პასუხების სტატუსი.

## რა არ შეცვლილა

- კონფიდენციალურობის რეჟიმი — MRI მონაცემები კვლავ მხოლოდ ბრაუზერშია, არასოდეს იგზავნება სერვერზე.
- სამედიცინო ნომრები (MRN), ექიმების სახელები, დაბადების თარიღი — არცერთი ვერ ჩანს ვებგვერდის კოდში.
- ენის გადართვა — ქართულ/ინგლისურ კონტრაქტი ხელუხლებელია; 244 თარგმანის გასაღები (143 ძველი + 101 ახალი) გადამოწმდა ერთეულ-ერთეულზე.

## ტექნიკური მახასიათებლები (მოკლედ)

- ყველა მძიმე ვიზუალური ბიბლიოთეკა (Plotly, vis-network, react-flow) იტვირთება მხოლოდ მაშინ, როცა შესაბამის გვერდს გახსნი — მთავარი bundle არ იზრდება.
- ყველა ახალი გვერდი დაცულია feature flag-ით; ემერჯენსი დროს შაკოს შეუძლია `viewer/lib/flags.ts`-ში ერთი მნიშვნელობის გადართვით გვერდი დამალოს.
- ჯერ ცოცხალი backend არ არის მიბმული — ყველა გვერდი მუშაობს დეტერმინისტული საცდელი მონაცემებით. ეს განზრახ — სტრუქტურა ჯერ უნდა გადამოწმდეს, შემდეგ ცოცხალი API ერთვება.
- ქართული ტექსტის ხარისხის შემოწმება ავტომატური ვერიფაიერშია ჩაშენებული: არცერთი დაგმობილი სიტყვა არ მეორდება ერთ აბზაცში, არცერთი ემ-დაში არ ფიქსირდება, რიცხვი 13 ცხრილში ციფრებითაა.

## შემდეგი ნაბიჯები (შაკოსთვის)

1. ლოკალურად გაუშვი `cd viewer && npm run dev`, შემდეგ ბრაუზერში გადახედე ოთხ ახალ გვერდს (`/ka/twin`, `/ka/drift`, `/ka/causal`, `/ka/simulate`) ქართულად და ინგლისურად.
2. დარწმუნდი, რომ ჰისტოგრამები რეალურად იხატება, რომ კვანძები ჩანს კაუზალურ გრაფიკზე, რომ react-flow-ის drag-and-drop მუშაობს.
3. გადაჭერი middleware.ts + proxy.ts კონფლიქტი (Phase 7.5-დან მოყოლებული პრობლემა), რომელიც ხელს უშლის `npm run build`-ის გავლას.
4. Vercel-ზე გაიარე preview deploy და ცოლს გაუგზავნე ლინკი.

ციფრული ტყუპის სტრუქტურა მზადაა. ცოცხალი ნახვა შაკოს smoke-სესიის შემდეგ ხდება.
