    (6) Transaction: open ONE transaction (autocommit off). In live mode, COMMIT once at the end after all tables processed; on any exception, ROLLBACK and surface. In dry-run mode, never write; ROLLBACK at the end.

    (7) Print a per-table summary: rows scanned, rows eligible (mirror), fields updated, fields blocked-by-redact, fields skipped (already translated). In dry-run, header clearly states "DRY RUN — no writes, $0 spend (test-mode stub)".

    Do NOT hardcode the connection string. Do NOT log full PHI text — truncate previews to ~60 chars. Use parameterized SQL only (no f-string interpolation of values into SQL).
