# Phase 6.1 Context — Bilingual Polish (i18n gap closure)

Polish/gap-closure phase derived from Phase 6 (Bilingual System i18n, closed 2026-05-21). Browser inspection of the live `/ka` locale on `viewer-sigma-two.vercel.app` surfaced three categories of un-translated content. This phase closes all three. The outcome: a fully Georgian `/ka` experience — UI chrome, enum labels, and existing data content.

## Locked Decisions

### D-01 — Three independent directions, one per plan
- **6.1-01 BRAIN panel i18n** — frontend, autonomous
- **6.1-02 ENUM label translation** — frontend, autonomous
- **6.1-03 DB content backfill** — Python script + Shako-supervised production run (checkpoint `autonomous: false`)

### D-02 — BRAIN panel uses a new `Manager` namespace
New namespace `Manager` in `viewer/messages/{en,ka}.json` (~28 keys, key-set equal — `verify_phase6` check_i18n_03 enforces leaf-count parity). Client components (`ActivityFeed`, `EmailIntent`, `InputBar`, `VoiceRecorder`) use `useTranslations('Manager')` from `next-intl`; the server component `BrainPanel.tsx` becomes async and uses `await getTranslations('Manager')`. `NextIntlClientProvider` is already mounted in `viewer/app/[locale]/layout.tsx` (06-03b) so client hooks have context. Translate all 5 BRAIN components (InputBar/VoiceRecorder are dormant/unmounted but cheap and may be re-mounted). Use ICU message args for `summarize()`/`relativeTime()` interpolation.

### D-03 — ENUM labels via dedicated namespaces with raw-value fallback
New namespaces `TimelineEventType` (8 values), `HypothesisStatus` (5), `HypothesisType` (drug_repurposing + others present), `ConfidenceLevel` (3). Render sites: `viewer/app/[locale]/timeline/page.tsx` (event_type tag + count chips), `viewer/app/[locale]/hypotheses/page.tsx` (status, type, confidence_level). **Fallback rule: unknown enum value → render raw string** (never crash on an unmapped value). Both are server components → `getTranslations`.

### D-04 — One-time LLM backfill of existing ka=en mirror rows (Shako-run)
New script `scripts/migrations/013_backfill_ka_translations.py` + runbook `013_runbook.md` (mirror 012 pattern). Selects rows where `col->>'en' = col->>'ka'` AND non-empty across the 6 JSONB columns (`aleksandra_timeline.{title,description}`, `hypotheses.{title,description}`, `therapies.{name,evidence_summary}`); calls `compose_bilingual` per field; UPDATEs JSONB to `{en, ka}`. Idempotent (en≠ka rows skipped). Reuses Phase 6 safety rails: `check_daily_budget` (fires inside compose_bilingual), defensive `redact_bilingual`, single transaction. `BILINGUAL_TEST_MODE=1` → dry-run preview ($0, no writes). Estimated ~$0.6–1.2 LLM. **Checkpoint:** Shako runs dry-run preview → live backfill → confirms counts, mirroring migration 012's 06-07 pattern.

### D-05 — Reuse, do not reinvent
- `compose_bilingual` — `scripts/communicator/bilingual.py`

### D-06 — Verification preserves Phase 6 invariants
`verify_phase6 --mode code-complete` must stay 11/11 PASS (key-set parity holds with new namespaces). `verify_phase4` 9/9 + `verify_phase5` 13/13 regression must stay GREEN. Build `cd viewer && npm run build` exits 0.
