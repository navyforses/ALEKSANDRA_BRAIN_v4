### Task 1 — `scripts/migrations/013_backfill_ka_translations.py` (commit `ceb189b`)

244-line backfill script. Behavior:
- Connects via `SUPABASE_DB_URL` (same env var as `weekly_brief.py`); exits cleanly if unset.
- Iterates 6 targets: `aleksandra_timeline.{title,description}`, `hypotheses.{title,description}`, `therapies.{name,evidence_summary}`.
- SELECTs rows where `col->>'en' = col->>'ka'` AND `en` non-empty (mirror rows from migration 012).

- UPDATEs `{col}` to `jsonb_build_object('en', new_en, 'ka', new_ka)` (parameterized SQL).
- Single `BEGIN/COMMIT` transaction; ROLLBACK on any exception or in `--dry-run`.
- `--dry-run` CLI + `BILINGUAL_TEST_MODE=1` env → zero writes, $0 spend, prints intended UPDATEs.
- Idempotent: re-runs report 0 eligible (rows where `en != ka` are skipped).
- Per-table summary printed at end (scanned / eligible / updated / blocked / skipped_translated).
