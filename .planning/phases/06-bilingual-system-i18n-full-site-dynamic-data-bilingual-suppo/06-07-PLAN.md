    2. **Confirm rollback artifacts exist:**
       ```

       ```

    3. **Set the env var** in the shell you will run the migration from:

    4. **Apply the migration:**
       ```

       ```

    5. **Run the 6 post-apply smoke checks** (from 012_runbook.md), appending each to post_apply_smoke.txt:
       ```
       psql "$SUPABASE_DB_URL" -c "SELECT pg_typeof(title), pg_typeof(description) FROM aleksandra_timeline LIMIT 1;" >> scripts/migrations/012_rollback/post_apply_smoke.txt
       psql "$SUPABASE_DB_URL" -c "SELECT pg_typeof(title), pg_typeof(description) FROM hypotheses LIMIT 1;" >> scripts/migrations/012_rollback/post_apply_smoke.txt
       psql "$SUPABASE_DB_URL" -c "SELECT pg_typeof(name), pg_typeof(evidence_summary) FROM therapies LIMIT 1;" >> scripts/migrations/012_rollback/post_apply_smoke.txt
       psql "$SUPABASE_DB_URL" -c "SELECT id, title->>'en' = title->>'ka' AS mirrored FROM aleksandra_timeline LIMIT 5;" >> scripts/migrations/012_rollback/post_apply_smoke.txt
       psql "$SUPABASE_DB_URL" -c "SELECT count(*) FROM aleksandra_timeline WHERE title->>'en' = title->>'ka';" >> scripts/migrations/012_rollback/post_apply_smoke.txt
       psql "$SUPABASE_DB_URL" -c "SELECT count(*) FROM aleksandra_timeline;" >> scripts/migrations/012_rollback/post_apply_smoke.txt
       ```
       Expected: every `pg_typeof` returns `jsonb`; every `mirrored` column is `t` (true); the two count(*) queries return the same number (I18N-09 mirror).

    6. **Capture POST-migration RLS policy snapshots:**
       ```
       psql "$SUPABASE_DB_URL" -c "\d aleksandra_timeline" > scripts/migrations/012_rollback/aleksandra_timeline.policies.post.txt
       psql "$SUPABASE_DB_URL" -c "\d hypotheses"          > scripts/migrations/012_rollback/hypotheses.policies.post.txt
       psql "$SUPABASE_DB_URL" -c "\d therapies"           > scripts/migrations/012_rollback/therapies.policies.post.txt
       psql "$SUPABASE_DB_URL" -c "\d briefs"              > scripts/migrations/012_rollback/briefs.policies.post.txt
       ```

    7. **Diff pre vs post policies** — must match (RLS preservation, threat T-06-02):
       ```
       diff <(grep -E "^Policies:" -A 50 scripts/migrations/012_rollback/aleksandra_timeline.policies.pre.txt) \
            <(grep -E "^Policies:" -A 50 scripts/migrations/012_rollback/aleksandra_timeline.policies.post.txt)
       # Expected: empty diff or only trivial whitespace differences
       ```
       Repeat for the other 3 tables.

    8. **If ANY smoke check fails or the policy diff shows a missing policy**, STOP and execute the rollback procedure from 012_runbook.md → `## Rollback procedure`. Do NOT type "approved" in that case — type "rollback executed" and surface the failure mode.

    9. After all 6 smoke checks pass and 4 policy diffs are clean, type **`approved`** (or `approved with notes: ...` if you observed anything unusual but the apply succeeded).
