# ახალი რედიზაინის UX და მონაცემთა ლოგიკა

რედიზაინი გადადის მძიმე command-center ვიზუალიდან მშვიდ, მინიმალისტურ კვლევით ნავიგატორზე. მთავარი მიზანია, მომხმარებელმა პირველივე ეკრანზე გაიგოს სამი რამ: **რა ვიცით**, **რა რჩება გაურკვეველი** და **რა უნდა ვკითხოთ ექიმს**.

## ძირითადი flow

| ნაბიჯი | UI ბლოკი | დანიშნულება |
|---|---|---|
| 1 | მტკიცებულება | მოკლე, წყაროზე მიბმული კვლევითი დასკვნა. |
| 2 | რისკი | შეზღუდვა, გაურკვევლობა ან clinical boundary. |
| 3 | ექიმთან კითხვა | უსაფრთხო next step, რომელიც არ არის სამედიცინო რჩევა. |

## მონაცემის მინიმალური ერთეული

ყოველი თემა წარმოდგენილია როგორც `TopicModel`: მოკლე პასუხი, evidence grade, მტკიცებულების bullets, uncertainty bullets, risk note, doctor questions და brief items. UI არ უნდა იყოს hardcoded card-ების გალერეა; ყველა გვერდი უნდა აშენდეს ერთიანი pattern-ით.

## ვიზუალური პრინციპები

| პრინციპი | განხორციელება |
|---|---|
| მშვიდი dark theme | ნაკლები glow, თხელი border, მუქი slate ფონები. |
| ქართული-first | მოკლე labels, sentence case, ზედმეტი uppercase-ის გარეშე. |
| პრაქტიკული | თითოეულ ბლოკს აქვს შემდეგი მოქმედება. |
| ეთიკური | research-only disclaimer ჩანს header/assistant/flow-ში. |
| სწრაფად გასაგები | მთავარი ეკრანი იკითხება 10 წამში, თემის ბრიფი მზადდება 2 წუთში. |

## Navigation

Sidebar მცირდება ძირითად მიმართულებებზე: მთავარი, მტკიცებულება, ჰიპოთეზები, თერაპიები, რუკა, ბრიფი. დამხმარე route-ები ინარჩუნებს მხარდაჭერას, მაგრამ UI-ში არ ქმნის გადატვირთვას.
