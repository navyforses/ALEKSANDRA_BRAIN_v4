# Mockup Implementation Findings

## Concept A — Clinical Command Center

The visual language is a dark, high-tech clinical research command center. It uses a deep navy background, neon cyan and violet accents, thin bordered glass panels, a fixed left navigation rail, a main dashboard area, and a fixed right AI assistant panel. The hero area contains a large ALEKSANDRA_BRAIN title and a glowing neural/brain network visual. The screen is dense but organized through KPI cards, pipeline cards, charts, and a timeline. The main implementation requirements are: dark shell, left rail, right assistant, glowing neural background, KPI metric cards, evidence-to-hypothesis-to-therapy pipeline, research charts, and safety boundary badges.

## Concept B — Family-Safe Research Journey

The visual language is a light, calm, family-facing portal. It uses soft white and pale blue/green backgrounds, rounded cards, a left sidebar, a right ask-the-brain assistant, and a central journey tracker. The copy is supportive and non-alarming. The main hierarchy is: reassuring hero headline, research and care-support journey steps, four practical cards, and a clear bottom safety banner stating that doctors decide treatment. The implementation requirements are: light shell, family-friendly sidebar, soft cards, journey stepper, plain-language assistant, therapy watchlist, questions for doctor, and explicit safety boundary.

## Direction from user correction

The user does not want only the existing site restyled. The frontend must visibly implement these generated mockup concepts as real screens/components, especially their layout architecture and visual feel.

## Concept C — Digital Twin Brain Lab

The visual language is an immersive dark medical imaging laboratory. It has a left icon rail, top role switcher, study selector, a large central translucent 3D brain visualization, a left MRI layer control panel, a right evidence-links panel, a hypothesis inspector, and a bottom scan timeline. The strongest visual requirements are: a brain visualization that dominates the page, layer toggles with confidence percentages, colored affected/preserved regions, evidence link cards connected to regions, role-based tabs, and a compact timeline scrubber. This should become the Brain page's primary architecture rather than a placeholder.

## Hybrid implementation target

The site should not choose only one concept. The practical target is: Home and family-facing sections follow Concept B; Dashboard, Hypotheses, and Therapies follow Concept A; Brain follows Concept C; Timeline combines Concept B's calm journey language with Concept A/C clinical event structure.
